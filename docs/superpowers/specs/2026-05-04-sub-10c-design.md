# Sub-Project 10c — Production infra + DR — Design Spec

**Date:** 2026-05-04
**Status:** Approved
**Predecessor:** [Sub-10b Deployment automation](2026-05-03-sub-10b-design.md)
**Target release:** `infra-v1.0.0`
**Decomposition note:** Sub-10 splits into 10a / 10b / 10c. 10a (`app-v1.0.0`) shipped production builds + observability + real LLM. 10b (`deploy-v1.0.0`) shipped one-environment-end-to-end deployment automation. 10c is the final sub-project: AD federation, multi-environment promotion, network/TLS validation against IDD v1.2 hostnames, backup automation + DR, secret rotation, auto-rollback, production Sentry DSN provisioning. After 10c, the system is operationally complete on the IDD-specified target hardware.

---

## Goal

Take the deployable system from Sub-10b and make it operationally complete: federated against AD, reachable at IDD's production hostnames over TLS, recoverable from backup, promotable across four environments (test → pre-prod → prod → DR), with secret rotation procedures and a documented disaster recovery runbook. No new app features — Sub-10c is pure ops/infra layer on top of the running stack.

## Non-goals

- **Active-passive or active-active HA** (log-shipping, Always On AGs, distributed cache) — deferred to Sub-10d or never. RTO is hours, not seconds.
- **Vault / Azure Key Vault / AWS Secrets Manager graduation** — deferred. `.env.<env>` on host with NTFS ACLs + secret-rotation runbook is the 10c answer.
- **SPNEGO / Kerberos SSO** — deferred. Read-only LDAP federation only.
- **Cert auto-provisioning** — operator-driven (AD CS / `win-acme` / manual import); 10c documents three paths but doesn't pick or automate one.
- **DNS auto-provisioning** — operator-driven; 10c documents validation procedure.
- **WAF / IP allowlisting** — deferred to whenever IDD requires it.
- **Automated Sentry org provisioning** — operator does the click-path in Sentry's web UI.
- **Automated DR drills** — quarterly drill is a runbook checklist, not a CI workflow.
- **Multi-tenant / multi-host clustering** — single host per environment.
- **AD writes from app** — federation stays read-only.
- **Federation against AD FS or Azure AD** — raw LDAP only per IDD.

## Scope decisions (locked during brainstorm)

| # | Decision | Notes |
|---|---|---|
| 1 | **IIS reverse proxy on host (TLS termination)** | ARR rewrites to localhost ports 4200/4201/5001/5002. Native to Windows Server 2022; AD-joined host can use AD CS auto-enrollment for certs. |
| 2 | **Per-env files (`.env.{test,preprod,prod,dr}`) + `-Environment` switch** | Default `prod` for Sub-10b backward compat. Per-env `deploy-history-${env}.tsv` audit trail. |
| 3 | **Keycloak LDAP user federation, read-only** | LDAPS to `cce.local:636` (or 389 fallback). Group-mapper translates AD security groups to Keycloak roles per `permissions.yaml`. SPNEGO documented as Sub-10c+ enhancement. |
| 4 | **Ola Hallengren + Windows Task Scheduler + off-host sync** | 7-day full / 7-day diff / 24h log retention; weekly `DBCC CHECKDB`; hourly off-host UNC sync via robocopy. |
| 5 | **Stay on `.env` files; document rotation runbook + canary check** | `validate-env.ps1` rejects placeholder values + known-leaked patterns. Vault graduation is Sub-10d+. |
| 6 | **`-AutoRollback` switch + per-env `AUTO_ROLLBACK` opt-in** | Default `true` for test/preprod, `false` for prod/dr. Recursion guard prevents infinite loop. |
| 7 | **Single host per env + documented DR rebuild from backups** | RTO ~hours, RPO ≤15 minutes. DR host stays cold until promoted. |

## Components

### Identity — Keycloak LDAP user federation

**Realm:** `cce` (existing). Sub-10c adds an LDAP **User Federation provider**.

**LDAP federation provider config:**

| Setting | Value |
|---|---|
| Vendor | Active Directory |
| Connection URL | `ldaps://${LDAP_HOST}:636` (with `ldap://${LDAP_HOST}:389` fallback per env-file knob) |
| Bind type | `simple` (service-account bind via DN + password) |
| Bind DN | `${LDAP_BIND_DN}` (e.g. `CN=cce-keycloak-svc,CN=Users,DC=cce,DC=local`) |
| Bind credential | `${LDAP_BIND_PASSWORD}` (env-file only) |
| Edit mode | `READ_ONLY` (no AD writes; one-way trust) |
| Users DN | `${LDAP_USERS_DN}` (e.g. `OU=Users,DC=cce,DC=local`) |
| Username LDAP attribute | `sAMAccountName` |
| RDN LDAP attribute | `cn` |
| UUID LDAP attribute | `objectGUID` |
| User object classes | `person, organizationalPerson, user` |
| Search scope | `Subtree` |
| Validate password policy | `true` |
| Trust email | `true` |
| Sync registrations | `false` |
| Import users | `true` |
| Connection pooling | `true` |
| Trust store SPI | `ldapsOnly` |

**Group → role mapping** (committed in `realm-cce-ldap-federation.json`):

| AD security group | Keycloak role | Backend permission |
|---|---|---|
| `CCE-Admins` | `cce-admin` | full content + admin |
| `CCE-Editors` | `cce-editor` | content edit / publish |
| `CCE-Reviewers` | `cce-reviewer` | content review / approve |
| `CCE-Experts` | `cce-expert` | expert-profile self-service |
| `CCE-Users` | `cce-user` | read-only public surfaces |

**Provisioning script** (`infra/keycloak/apply-realm.ps1`):
- Reads `KEYCLOAK_AUTHORITY`, `KEYCLOAK_ADMIN_USER`, `KEYCLOAK_ADMIN_PASSWORD`, `LDAP_HOST`, `LDAP_BIND_DN`, `LDAP_BIND_PASSWORD`, `LDAP_USERS_DN`, `LDAP_GROUPS_DN` from `.env.<env>`.
- Authenticates against Keycloak admin REST API.
- POSTs the realm-import JSON parameter-substituted; PATCHes existing federation provider on re-run (idempotent).
- Triggers an initial user sync.
- Logs to `C:\ProgramData\CCE\logs\keycloak-apply-<UTC>.log`.

### Network — IIS reverse proxy + TLS

**IIS sites (one per IDD hostname):**

| IIS site | Hostname binding | Backend |
|---|---|---|
| `CCE-ext` | `CCE-ext` | `http://localhost:4200` |
| `CCE-admin-Panel` | `CCE-admin-Panel` | `http://localhost:4201` |
| `api.CCE` | `api.CCE` | `http://localhost:5001` |
| `Api.CCE-admin-Panel` | `Api.CCE-admin-Panel` | `http://localhost:5002` |

(IDD's "port 433" treated as 443 per session memory.)

**`web.config.template`** rewrite rule (parameterized by `BACKEND_PORT`):

```xml
<rewrite>
  <rules>
    <rule name="ProxyToBackend" stopProcessing="true">
      <match url="^(.*)$" />
      <action type="Rewrite" url="http://localhost:{BACKEND_PORT}/{R:1}" />
      <serverVariables>
        <set name="HTTP_X_FORWARDED_HOST" value="{HTTP_HOST}" />
        <set name="HTTP_X_FORWARDED_PROTO" value="https" />
        <set name="HTTP_X_FORWARDED_FOR" value="{REMOTE_ADDR}" />
      </serverVariables>
    </rule>
  </rules>
</rewrite>
<system.webServer>
  <httpProtocol>
    <customHeaders>
      <add name="Strict-Transport-Security" value="max-age=31536000; includeSubDomains" />
      <add name="X-Content-Type-Options" value="nosniff" />
      <add name="X-Frame-Options" value="SAMEORIGIN" />
    </customHeaders>
  </httpProtocol>
  <serverRuntime appConcurrentRequestLimit="5000" />
</system.webServer>
```

(`X-Forwarded-*` headers preserved so backend logs the real client IP. SSE-friendly: no response buffering — `appConcurrentRequestLimit=5000` allows long-held streams.)

**Cert lifecycle:** Sub-10c assumes operator-provided cert. Three documented paths:
1. AD CS auto-enrollment on AD-joined host.
2. `win-acme` against Let's Encrypt (DNS-01 challenge for non-public-facing hosts).
3. Manually procured cert imported via `Import-PfxCertificate`.

Cert thumbprint or PFX path goes into `.env.<env>`:
```bash
IIS_CERT_THUMBPRINT=A1B2C3...
# OR
IIS_CERT_PFX_PATH=C:\ProgramData\CCE\certs\cce-prod.pfx
IIS_CERT_PFX_PASSWORD=<set-me>
```

**`Configure-IISSites.ps1`:**
- Reads env-file via `-Environment` switch.
- Ensures IIS + URL Rewrite + ARR features (calls `Install-ARRPrereqs.ps1` if missing).
- Creates/updates 4 sites with hostname bindings, port 443, named cert, ARR `web.config` (rendered from template).
- Enables global ARR proxy.
- Restarts IIS at end. Idempotent.

**Smoke probe extension** (`deploy/smoke.ps1`):
- New env-aware mode: when `-Environment <env>` passed, probes 4 IDD hostnames over HTTPS instead of localhost.
- `-AllowSelfSignedCert` for test/preprod with internal CAs.

### Backup — Ola Hallengren + Task Scheduler

**Tool:** Ola Hallengren's [SQL Server Maintenance Solution](https://ola.hallengren.com/) v2024.x. Free, single-script install in `master`. Procs used: `DatabaseBackup`, `DatabaseIntegrityCheck`, `IndexOptimize`.

**Backup schedule:**

| Task | Frequency | Backup type | Retention | Destination |
|---|---|---|---|---|
| `CCE-Backup-Full` | Daily 02:00 local | `FULL` of `CCE` DB | 7 days | `D:\CCEBackups\FULL\` |
| `CCE-Backup-Diff` | Every 6 hours | `DIFF` of `CCE` DB | 7 days (rolling with full chain) | `D:\CCEBackups\DIFF\` |
| `CCE-Backup-Log` | Every 15 minutes | `LOG` of `CCE` DB | 24 hours rolling | `D:\CCEBackups\LOG\` |
| `CCE-Backup-IntegrityCheck` | Weekly Sunday 03:00 | `DBCC CHECKDB` | logs only | `C:\ProgramData\CCE\logs\integrity-<UTC>.log` |
| `CCE-Backup-Sync-OffHost` | Hourly | `robocopy /MIR` to UNC | 30-day retention at destination | `\\${BACKUP_UNC_HOST}\${BACKUP_UNC_SHARE}\<env>\` |

**Recovery model:** `FULL` (required for log backups; gives 15-minute RPO).

**Files:**
- **`infra/backup/ola-hallengren-install.sql`** — script committed verbatim (with version + checksum header).
- **`infra/backup/scheduled-tasks.xml`** — schtasks XML for the 5 tasks. Tasks run as `cce-sqlbackup-svc` service account (operator provisions).
- **`infra/backup/Sync-OffHost.ps1`** — `robocopy /MIR /Z /R:3 /W:10 /LOG+:...`. UNC auth via `cmdkey` setup (one-time during host setup).
- **`infra/backup/Restore-FromBackup.ps1`** — operator-driven helper. Takes `-FullBackup`, `-DiffBackup` (optional), `-LogBackups <path-list>`, `-TargetDb`. Sequence: full `NORECOVERY` → diff `NORECOVERY` → logs `NORECOVERY` → final `RECOVERY`. Verifies migration history matches expected. Refuses to restore over live `CCE` DB unless `-Force`.
- **`infra/backup/Test-BackupChain.ps1`** — queries `master.dbo.CommandLog` for last 24h `DatabaseBackup` runs; reports failures, last full backup time, log-chain gaps. Called by `deploy.ps1` post-deploy step (warn-only — doesn't fail deploy).

### Secret rotation runbook (`docs/runbooks/secret-rotation.md`)

| Secret | Rotation procedure | Frequency |
|---|---|---|
| `INFRA_SQL` password | New SQL login → update `.env.<env>` → `deploy.ps1` → drop old login | Quarterly |
| `LDAP_BIND_PASSWORD` | Reset AD service account → update `.env.<env>` → `deploy.ps1` → `apply-realm.ps1` to re-validate | Quarterly |
| `ANTHROPIC_API_KEY` | New key in Anthropic console → update → `deploy.ps1` → revoke old | On compromise OR semi-annually |
| `KEYCLOAK_ADMIN_PASSWORD` | Update via Keycloak admin UI → update `.env.<env>` | On compromise OR annually |
| `SENTRY_DSN` | Sentry project regen DSN → update `.env.<env>` → `deploy.ps1` | On compromise only |
| `BACKUP_UNC_PASSWORD` | Reset on file server → update `.env.<env>` → `cmdkey /delete` + re-add | On compromise OR annually |
| `IIS_CERT_PFX_PASSWORD` | Per-cert; rotated when cert renewed | At cert renewal |
| `CCE_GHCR_TOKEN` | GitHub PAT regen → update `.env.<env>` | Annually |

Each row has a "verify rotation worked" step.

### Canary integrity check (`deploy/validate-env.ps1`)

Standalone-callable; called by `deploy.ps1` step 2 after env-file parsing. Checks:
- Placeholder values still present: `<set-me>`, `<github-org-or-user>`, `Strong!Passw0rd`, `changeme`, `example.com`.
- Committed-secret canaries: known example credentials (e.g., `AKIAIOSFODNN7EXAMPLE`).
- Suspicious whitespace: trailing CR or BOM that breaks `env_file:` parsing.

Exits non-zero on any failure with a precise message.

### Auto-rollback (`deploy.ps1` extension)

**`-AutoRollback` switch:**
- When passed, smoke-probe failure (Step 8) triggers immediate `rollback.ps1` invocation.
- When omitted, env-file's `AUTO_ROLLBACK` value is the default.
- Explicit `-NoAutoRollback` overrides env-file `AUTO_ROLLBACK=true`.

**Rollback target resolution:**
- Most recent `OK` row in `deploy-history-${env}.tsv` whose tag differs from `CCE_IMAGE_TAG`.
- If no prior `OK` row exists, auto-rollback skipped with a clear log line.

**Auto-rollback flow (Step 8 modified):**

```
Smoke probe → FAIL.
Auto-rollback enabled?
  → yes: resolve previous tag from deploy-history-${env}.tsv → "<prev-tag>".
  → Log: "Smoke failed; auto-rolling back to <prev-tag>".
  → Invoke rollback.ps1 -ToTag <prev-tag> -EnvFile <resolved>.
    → rollback.ps1 swaps tag in .env.<env>, calls deploy.ps1 -Recursive.
    → If THAT smoke probe fails → exit non-zero, "rollback also failed; manual intervention required".
  → Both succeed → log "Auto-rollback complete; live tag is <prev-tag>".
deploy-history-${env}.tsv records both rows.
```

**Recursion guard:** `deploy.ps1 -Recursive` (set by `rollback.ps1` only) suppresses auto-rollback inside the nested call.

**Sentry breadcrumb:** auto-rollback sends a Sentry event tagged `deploy.auto_rollback` with failed tag, rolled-back-to tag, and probe-failure detail.

### Sentry production DSN

**Provisioning** (operator one-time per environment):
1. Create Sentry project (`cce-prod`, `cce-preprod`, `cce-test`) — manual click-path in Sentry web UI.
2. Paste per-env DSN into `.env.<env>` as `SENTRY_DSN=https://<key>@<sentry-host>/<project-id>`.
3. Tag environment: `SENTRY_ENVIRONMENT=production` (or `preprod` / `test` / `dr`).
4. Set release tag: new env-var `SENTRY_RELEASE=${CCE_IMAGE_TAG}`.
5. Configure Sentry alert rules in UI (manual; runbook documents recommended baseline).

**Backend code change** (Phase 04):

```cs
.WriteTo.Sentry(o => {
    o.Dsn = sentryDsn;
    o.Environment = ctx.Configuration["SENTRY_ENVIRONMENT"] ?? "unknown";   // NEW
    o.Release = ctx.Configuration["SENTRY_RELEASE"];                       // NEW
    o.MinimumEventLevel = LogEventLevel.Warning;
});
```

New test in `LoggingExtensionsTests` asserting `SENTRY_ENVIRONMENT` + `SENTRY_RELEASE` pass through.

### DR procedure

**DR-host pre-provisioning** (one-time, before disaster) — `infra/dns-tls/dr-provisioning-checklist.md`:

- [ ] Windows Server 2022, domain-joined to `cce.local`.
- [ ] Same hardware shape as prod (Intel Xeon Gold 6138).
- [ ] Docker installed.
- [ ] SQL Server installed, AD-auth enabled.
- [ ] Redis installed.
- [ ] Keycloak installed; LDAP federation realm imported (via `apply-realm.ps1 -Environment dr`).
- [ ] IIS + URL Rewrite + ARR; sites configured (via `Configure-IISSites.ps1 -Environment dr`).
- [ ] DR-environment cert in IIS cert store.
- [ ] `C:\ProgramData\CCE\.env.dr` populated.
- [ ] `cmdkey /add:${BACKUP_UNC_HOST}` cached.
- [ ] `D:\CCEBackups\restored\` exists.
- [ ] Ola Hallengren installed (no scheduled tasks active while cold).
- [ ] DR DNS records exist with low TTL but not yet pointing to DR.

**DR promotion runbook** (`docs/runbooks/dr-promotion.md`):

1. **Decision** — operator + ops lead confirm DR promotion. Record in incident log.
2. **Fetch latest backup chain** — `robocopy` from off-host UNC to DR's `D:\CCEBackups\restored\`.
3. **Restore DB** — `Restore-FromBackup.ps1 -Force -TargetDb CCE`.
4. **Deploy apps** — `deploy.ps1 -Environment dr`.
5. **Verify DR host green** — `smoke.ps1 -Environment dr -Timeout 90`.
6. **DNS swap** — repoint 4 hostnames at DR host IP (manual, operator-managed DNS).
7. **End-to-end validation** — from outside, `Test-NetConnection -ComputerName CCE-ext -Port 443`; smoke `https://CCE-ext/` from real client.
8. **Communicate** — status page / customer notification per ops.

**DR-back-to-prod (failback):** reverse procedure documented in same runbook.

**RTO:** ~hours. **RPO:** ≤15 minutes (log-backup interval).

## Error handling

| Failure | Detection | Behaviour |
|---|---|---|
| Required env-key missing | `validate-env.ps1` step | Abort before docker. Print missing keys + canary patterns. Exit non-zero. |
| Placeholder value detected | `validate-env.ps1` step | Abort with precise list of canary matches. Exit non-zero. |
| LDAP bind fails | `apply-realm.ps1` test-connection step | Abort with LDAP error code + suggested fixes (DNS, cert validation, bind-DN typo). |
| IIS site provisioning fails | `Configure-IISSites.ps1` | Roll back partial config (delete created sites), exit non-zero with `Get-IISSite` diff. |
| Cert thumbprint not in store | `Configure-IISSites.ps1` | Abort before binding; print available thumbprints. |
| Backup task creation fails | `schtasks /create` exit code | Abort during host setup. Operator runs `Get-ScheduledTask` to inspect. |
| Off-host UNC unreachable | `Sync-OffHost.ps1` robocopy exit | Warn (not abort). Logs to backup-sync log. Operator alerted via Sentry breadcrumb. |
| Restore fails | `Restore-FromBackup.ps1` | Abort. Refuses to leave DB in `RESTORING` state — runs final `RESTORE WITH RECOVERY` only on success path. |
| Auto-rollback infinite loop | `-Recursive` guard | Inner deploy refuses to auto-rollback. Outer fails non-zero with "manual intervention required". |
| Smoke probe fails post-rollback | smoke.ps1 in recursive deploy.ps1 | Exit non-zero. Both deploys recorded in deploy-history-${env}.tsv. |
| Sentry DSN typo | None at deploy time (Sentry sink swallows) | `validate-env.ps1` does a non-blocking warn for prod/preprod if DSN is empty. |
| DNS not resolving DR hostname during failover | manual operator check (Step 7) | Operator forces DNS refresh; documented in runbook. |
