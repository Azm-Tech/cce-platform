## 1. Goal

Layer the deep Knowledge Maps UX on top of the Sub-6 Phase 9 skeleton. After Sub-7, public users open a map, see its nodes laid out as an interactive graph, click any node to read its detail, search and filter to focus on concepts of interest, hold multiple maps open in tabs, and export selections in four formats (PDF, PNG, SVG, JSON). All read-only — Sub-7 ships zero write endpoints.

## 4. Out of scope (deferred)

- **Related maps suggestions** — backend doesn't expose `/related`; deferred to v0.2.0 once usage data informs a real recommender.
- **Side-by-side map comparison** (vs tabs) — overkill for v0.1.0; "Compare with…" toggle deferred to polish.
- **Editing maps / nodes / edges** — Sub-7 is read-only. Editing lives in admin-cms workflows (Sub-5) if/when added.
- **Graph layout algorithms** — backend supplies `LayoutX/LayoutY` per node; Cytoscape uses `preset` layout. No client-side cose / dagre / breadthfirst.
- **Vector PDF (text-as-text)** — PDF export wraps a high-DPI PNG inside a PDF page. Vector PDF via `svg2pdf.js` doubles bundle weight and has font-fidelity issues with ar text.
- **SVG export plugin in initial bundle** — `cytoscape-svg` lazy-loaded with the rest of Cytoscape on the Maps route only.

### 5.3 State management — `MapViewerStore`

A single Angular service provided at the route level. All state lives in signals:

```ts
interface ViewerTab {
  id: string;
  metadata: KnowledgeMap;        // from getMap(id)
  nodes: KnowledgeMapNode[];     // from getNodes(id)
  edges: KnowledgeMapEdge[];     // from getEdges(id)
  loadedAt: number;
}

@Injectable()
export class MapViewerStore {
  private readonly _tabsById = signal(new Map<string, ViewerTab>());
  private readonly _activeId = signal<string | null>(null);
  private readonly _selectedNodeId = signal<string | null>(null);
  private readonly _searchTerm = signal('');
  private readonly _filters = signal(new Set<NodeType>());      // empty = all
  private readonly _viewMode = signal<'graph' | 'list'>('graph');
  private readonly _selection = signal(new Set<string>());      // for export

  readonly openTabs = computed(() => Array.from(this._tabsById().values()));
  readonly activeTab = computed(() => {
    const id = this._activeId();
    return id ? (this._tabsById().get(id) ?? null) : null;
  });
  readonly selectedNode = computed<KnowledgeMapNode | null>(() => { /* lookup */ });
  readonly matchedIds = computed<Set<string>>(() => { /* search + filter */ });
  readonly dimmedIds = computed<Set<string>>(() => { /* allIds - matchedIds */ });

  async openTab(id: string): Promise<void> { /* load and switch */ }
  closeTab(id: string): void { /* remove + fallback active */ }
  setActive(id: string): void { ... }
  selectNode(id: string | null): void { ... }
  setSearch(term: string): void { ... }
  setFilter(types: NodeType[]): void { ... }
  setViewMode(mode: 'graph' | 'list'): void { ... }
  setSelection(ids: Set<string>): void { ... }
}
```

### 5.4 URL state

`/knowledge-maps/:id?open=a,b,c&node=:nodeId&q=carbon&type=Technology&view=graph`

- `:id` — currently active map (canonical route param).
- `?open=` — comma-separated other open tabs (excluding active).
- `?node=` — selected node id; cleared on close.
- `?q=` — search term.
- `?type=` — comma-separated NodeType filter (empty = all).
- `?view=` — `graph` (default) or `list`.

All changes via `router.navigate(...)` with `queryParamsHandling: 'merge'` and `replaceUrl: true` so browser-back stays sane.

## 8. UX decisions baked in (from brainstorming)

| Decision | Choice | Rationale |
|---|---|---|
| MVP scope | Full exploration suite (option C) minus deferred items | Stretch + good demo surface; deferred items (related maps, side-by-side) recouped via tabs |
| Graph library | Cytoscape.js | Battle-tested for knowledge graphs; rich interaction; 6 layout algorithms (we use `preset`) |
| Layout | Server-driven (`LayoutX/Y` from backend) | No client-side layout math; instant render; reliable across reloads |
| Detail panel | Persistent right-side rail (desktop) / bottom sheet (mobile) | Keeps graph visible while reading; standard map-detail pattern |
| Multi-map | Tabs (no split panes) | 95% of "compare" workflow served by toggling tabs; defers complexity |
| Related maps | **Deferred to v0.2.0** | No backend signal; client heuristic too noisy |
| Search/filter | Highlight + dim | Convention in scientific knowledge tools (Reactome, Wikipathways) |
| Export | PDF + PNG + SVG + JSON | Slides + archive + vector + programmatic — all 4 paid lazy on the route |
| RTL | Mirror node x-coordinates when locale === 'ar' | Graph reflects horizontally to match RTL chrome |
| Mobile | Bottom sheet replaces side panel | Universal mobile pattern (Google Maps, Apple Maps) |
| Accessibility | View toggle: graph ↔ list | W3C-recommended pattern for SVG-style diagrams; useful for sighted users too |

## 9. Critical user flows

| Flow | Steps | Phase |
|---|---|---|
| Open a map | List page → click card → `/knowledge-maps/:id` → graph renders, all nodes visible | 7.2 |
| Read a node | Click node → side panel slides in → name + description + outbound edges list → click an edge target → re-renders panel for that node | 7.3 |
| Search a concept | Type in search bar (debounced) → matching nodes light up, others dim → click a match → centers + selects | 7.4 |
| Filter by type | Click "Technology" chip → only Technology nodes lit, others dim → toggle off → restores | 7.4 |
| Open second map | List page → click second card → new tab appears in active tab strip → first tab data preserved | 7.5 |
| Close a tab | Click `×` on a tab → tab removed → if was active, fall back to last remaining | 7.5 |
| Export selection | Multi-select nodes (rubber band or shift-click) → click "Export" → pick format → file downloads | 7.6 |
| Switch to list view | Click "List view" toggle → Cytoscape unmounts, structured `<ul>` renders → keyboard tab through | 7.7 |
| Switch locale to ar | LocaleSwitcher → Cytoscape positions mirror, panel + tabs flip RTL, list view re-renders ar | 7.8 |

## 10. Error handling

- **Map 404:** GraphCanvas hidden; not-found block with link back to `/knowledge-maps`.
- **Nodes/edges fetch error:** Graph shows error banner with retry button; tab stays open.
- **Cytoscape lazy-load failure** (rare CDN issue): falls back to ListView automatically; toast warns "Graph viewer unavailable, showing list view".
- **Export failure:** Toast with `errors.<kind>`; menu stays open so user can pick a different format.
