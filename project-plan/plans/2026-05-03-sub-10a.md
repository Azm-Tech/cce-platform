# CCE Sub-Project 10a — App productionization — Implementation Plan (Master)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan phase-by-phase. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Take the four shipped applications (`Api.External`, `Api.Internal`, `web-portal`, `admin-cms`) from "passes tests on a developer laptop" to "production-quality builds with real observability and a real LLM". Sub-10a is app-layer only; deployment automation is Sub-10b, infrastructure (AD, multi-env, backup, IDD DNS/LB validation) is Sub-10c.

**Architecture:** Four multistage Dockerfiles (Linux-based) build production images for the two .NET 8 APIs and the two Angular SPAs (nginx-served). New `CCE.Api.Common.Observability` namespace hosts `UseCceSerilog` (console JSON + rolling file + Sentry sink, all gated on env-vars) and `UseCcePrometheus` (`/metrics` endpoint via prom-net). New `CCE.Infrastructure.Assistant.AnthropicSmartAssistantClient` implements the existing `ISmartAssistantClient` interface against the `Anthropic.SDK` NuGet; `AssistantClientFactory` picks stub vs Anthropic from `Assistant:Provider` config. Citation search is RAG-lite: token-overlap scoring against seeded `Resources` + `KnowledgeMapNodes`. Two new GitHub Actions workflows run Lighthouse against `/knowledge-maps/:id` and axe-core against `/interactive-city` + `/assistant`.

**Tech Stack:** .NET 8 minimal APIs + MediatR (existing); `Anthropic.SDK` NuGet (NEW); Serilog 4.1.0 + Sentry.Serilog 4.13.0 + prometheus-net 8.2.1 (already in `Directory.Packages.props`, newly wired); Docker multistage builds against `mcr.microsoft.com/dotnet/{sdk,aspnet}:8.0` and `node:22-alpine` + `nginx:alpine`; `treosh/lighthouse-ci-action@v11`; `@axe-core/playwright` (already used by `web-portal-e2e`).

**Spec reference:** [`../specs/2026-05-03-sub-10a-design.md`](../specs/2026-05-03-sub-10a-design.md) — 12 sections, 4-phase plan, ~18 tasks.

## Global conventions

### Working directory

All paths relative to repo root `/Users/m/CCE/`. Frontend commands run from `frontend/`; backend commands from `backend/`. The plan calls out `cd <dir>` where needed.

### Git workflow

- One commit per task (atomic, reviewable history).
- Conventional commit format: `<type>(<scope>): <subject>`. Scopes for Sub-10a: `feat(api-common)`, `feat(infrastructure)`, `feat(api-external)`, `feat(api-internal)`, `feat(web-portal)`, `feat(admin-cms)`, `chore(docker)`, `chore(ci)`, `docs(sub-10a)`.
- Commit with `git -c commit.gpgsign=false commit -m "..."` if signing is set up. No `--no-verify`.
- Gitleaks pre-commit hook stays active.

### TDD discipline (per ADR-0007 + Sub-6 through Sub-9 pattern)

**Strict TDD** for:
- `CitationSearch` token-overlap scoring + locale field selection.
- `AssistantClientFactory` provider/key resolution.
- `AnthropicSmartAssistantClient` event-streaming behaviour (mocked SDK).

**Test-after** for:
- `LoggingExtensions` / `PrometheusExtensions` — host wiring is best verified end-to-end via the integration test project.
- Dockerfiles — verified by CI build success + container smoke probes.

### Patterns inherited from Sub-6 through Sub-9

- **Result<T> + toFeatureError** — frontend pattern, unchanged. Sub-10a doesn't add new frontend Result-returning methods.
- **`ISmartAssistantClient.StreamAsync(messages, locale, ct) → IAsyncEnumerable<SseEvent>`** — Sub-9 contract is preserved verbatim. The Anthropic implementation is interchangeable with the stub.
- **`SseEvent` discriminated union** — `TextEvent` / `CitationEvent(CitationDto)` / `DoneEvent` / `ErrorEvent(ErrorPayload)` — defined in `CCE.Application.Assistant`. Sub-10a yields the same types from the new client.
- **Existing `CorrelationIdMiddleware`** — already sets `Activity.Current?.SetBaggageItem("correlation_id", ...)`. Serilog enricher reads from there.

### Test running

Frontend:
```bash
cd frontend && ./node_modules/.bin/nx test web-portal --watch=false
```

Backend (Application + Infrastructure unit tests, no external services):
```bash
cd backend && dotnet test tests/CCE.Application.Tests/ tests/CCE.Infrastructure.Tests/
```

Backend integration tests need Keycloak + SQL containers; not part of Sub-10a's primary gate but should be checked once at close-out.

E2E:
```bash
cd frontend && ./node_modules/.bin/nx e2e web-portal-e2e
```

Build / lint:
```bash
cd frontend && ./node_modules/.bin/nx run web-portal:lint && ./node_modules/.bin/nx build web-portal
cd backend && dotnet build
```

Docker builds (smoke):
```bash
docker build -t cce-api-external:dev -f backend/src/CCE.Api.External/Dockerfile backend/
docker build -t cce-api-internal:dev -f backend/src/CCE.Api.Internal/Dockerfile backend/
docker build -t cce-web-portal:dev -f frontend/apps/web-portal/Dockerfile .
docker build -t cce-admin-cms:dev -f frontend/apps/admin-cms/Dockerfile .
docker compose -f docker-compose.prod.yml up -d --build
curl -fsS http://localhost:5001/health  # expects 200
curl -fsS http://localhost:5001/metrics | head -3  # expects prom-format
curl -fsS http://localhost:4200/ | head -10  # expects index.html
docker compose -f docker-compose.prod.yml down
```

### Definition of done (per phase)

- All new files have at least one Jest or xunit test (where the file represents testable behaviour; Dockerfiles are verified via build + smoke probe instead).
- Frontend: `nx test web-portal --watch=false` passes locally.
- Frontend: `nx run web-portal:lint` passes (zero new warnings).
- Frontend: `nx build web-portal` and `nx build admin-cms` succeed (production config).
- Backend: `dotnet build` and `dotnet test tests/CCE.Application.Tests/` pass locally.
- New env-vars documented in the per-phase plan.
- One commit per task; conventional message; co-author trailer.

## Risks + mitigations

| Risk | Mitigation |
|---|---|
| `Anthropic.SDK` API surface changes between versions | Pin to a specific minor; document in ADR-0051. The `IAsyncEnumerable<SseEvent>` boundary insulates Sub-9 frontend from any swap. |
| Anthropic streaming events don't map cleanly to `SseEvent` | The plan includes a small adapter layer (`AnthropicSmartAssistantClient`) that's the only file that needs to know the SDK shape. Frontend stays unchanged. |
| Citation search returns irrelevant rows for Arabic queries | Locale-aware field selection (`TitleAr` for `locale=ar`, `TitleEn` for `locale=en`); tokens lowercased; substring fallback if Jaccard score is zero. |
| Lighthouse CI is flaky on the seeded route | The Lighthouse run depends on a populated DB. Workflow seeds via the existing `CCE.Seeder` console app before launching `npx serve`. |
| Docker build downloads gigabytes of base images on every CI run | Use `docker/build-push-action` with GH cache; persistent layer cache between runs. |
| Sentry sink fires on every test run because `SENTRY_DSN` accidentally leaks | The sink is gated on `SENTRY_DSN` env-var being set AND non-empty. CI doesn't set it. |
| File sink fills the container disk in long-running prod | Daily rolling + `RetainedDays` limit (default 7). Sub-10c will revisit retention against IDD policy. |
