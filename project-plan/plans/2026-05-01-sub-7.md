# CCE Sub-Project 07 — Knowledge Maps full UX — Implementation Plan (Master)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan phase-by-phase. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Layer the deep Knowledge Maps UX on top of the Sub-6 Phase 9 skeleton. After Sub-7, public users open a map at `/knowledge-maps/:id`, see its nodes laid out as an interactive Cytoscape graph (server-driven `LayoutX/Y` positions), click any node to read its detail in a side panel, search and filter to focus on concepts, hold multiple maps open in tabs, switch to a list view for accessibility, and export selections in PDF / PNG / SVG / JSON. Read-only — Sub-7 ships zero write endpoints. Backend unchanged.

**Architecture:** All new surface lives under `apps/web-portal/src/app/features/knowledge-maps/`. A single `MapViewerStore` (signals-first) owns tabs / selection / search / filters / view-mode. Heavy dependencies (`cytoscape`, `cytoscape-svg`, `jspdf`) are dynamically imported inside `cytoscape-loader.ts` so they ship only on the `/knowledge-maps/:id` lazy chunk — initial bundle untouched. URL state captures tabs/selection/search/filter/view-mode for deep-linking.

**Tech Stack:** Angular 19 standalone, signals-first, Material 18, ngx-translate ar/en + RTL, Cytoscape.js + cytoscape-svg + jsPDF (lazy), Jest (unit), Playwright + `@axe-core/playwright` (E2E + a11y). All in `frontend/package.json`.

**Spec reference:** [`../specs/2026-05-01-sub-7-design.md`](../specs/2026-05-01-sub-7-design.md) — 14 sections, 9-phase plan, ~38 tasks. Sub-8 (Interactive City) and Sub-9 (Smart Assistant) decomposed during brainstorming and queued for separate cycles.

### Lazy loading boundary

The `/knowledge-maps/:id` route uses `loadComponent` to defer `MapViewerPage`. Inside `MapViewerPage`, `cytoscape-loader.ts` performs `await import('cytoscape')` on first use; `cytoscape-svg` and `jspdf` are imported only when the user picks SVG or PDF in the export menu. **Initial web-portal bundle stays unchanged from Sub-6.**

### URL-state contract

`/knowledge-maps/:id?open=a,b,c&node=:nodeId&q=carbon&type=Technology&view=graph`

| Param | Meaning | Default |
|---|---|---|
| `:id` (route) | Active tab map id | required |
| `?open=` | Comma-separated other open tabs | empty |
| `?node=` | Selected node id | none |
| `?q=` | Search term | empty |
| `?type=` | Comma-separated NodeType filter | empty (=all) |
| `?view=` | `graph` or `list` | `graph` |

All updates use `router.navigate(...)` with `queryParamsHandling: 'merge'` and `replaceUrl: true`.

### RTL strategy

When `LocaleService.locale() === 'ar'`, all node `LayoutX` values are negated (`x → -x`) before being passed to Cytoscape. The `LayoutY` values are unchanged. An `effect()` inside `GraphCanvasComponent` watches the locale signal and re-positions live when the user toggles mid-session, preserving zoom/pan. The chrome (TabsBar, NodeDetailPanel, SearchAndFilters) flips automatically via standard CSS RTL.

### Accessibility strategy

Each map has two presentations of the same data:
1. `GraphCanvasComponent` — Cytoscape canvas (visual / sighted users).
2. `ListViewComponent` — nested `<ul>` grouped by NodeType (keyboard / screen-reader users).

The `MapViewerStore.viewMode` signal toggles between them. Both components write to the same `selectedNodeId` action, so toggling preserves selection. URL `?view=` captures the choice. WCAG meaningful-content tests run in Phase 8 via axe-core.
