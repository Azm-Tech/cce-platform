```bash
  --bundler=esbuild \
```

Expected: prints a "Successfully created the workspace" message. Takes 2–5 minutes (large package install).

## Task 9.3: Generate `admin-cms` app

**Files:**

**Rationale:** Mirror of web-portal but at port 4201, separate routing, will get OIDC login wiring in Phase 12.

```bash

  --strict=true \

  --unitTestRunner=jest \

cd ..
```

Expected: prints "CREATE apps/admin-cms/...". Generates ~30 files.

- [ ] **Step 2: Set web-portal port to 4200 and admin-cms port to 4201**

The Nx Angular generator may not respect `--port` on Nx 20.5 — the dev server config lives in `apps/<app>/project.json` under `targets.serve.options.port`. Verify and patch if needed:

```bash
grep -l '"port"' frontend/apps/web-portal/project.json frontend/apps/admin-cms/project.json
```

If `web-portal/project.json` has no port (Nx default 4200 is fine) and `admin-cms/project.json` doesn't have `4201`, set them. The exact location is `targets.serve.options.port`. Use `jq` or hand-edit:

```bash
# Read current ports
node -e "const p = require('./frontend/apps/web-portal/project.json'); console.log('web-portal port:', p.targets?.serve?.options?.port ?? 'default-4200');"
node -e "const p = require('./frontend/apps/admin-cms/project.json'); console.log('admin-cms port:', p.targets?.serve?.options?.port ?? 'default-4200');"
```

If admin-cms port is missing or 4200, hand-edit `frontend/apps/admin-cms/project.json` to set `targets.serve.options.port: 4201`.

## Task 9.4: Install runtime packages — Material, Bootstrap grid, ngx-translate, OIDC client

**Files:**

- Modify: `frontend/package.json`
- Modify: `frontend/pnpm-lock.yaml`
- Possibly modify: `frontend/apps/*/src/styles.scss` (add base imports)

**Rationale:** Foundation needs:

- `@angular/material` 18.2.x — components + theming
- `@angular/cdk` 18.2.x — peer of Material
- `bootstrap` 5.3.x — for grid/utilities ONLY (not the components)
- `@ngx-translate/core` 16.x + `@ngx-translate/http-loader` 16.x — i18n
- `angular-auth-oidc-client` 18.x — Keycloak code-flow + PKCE

- [ ] **Step 1: Install packages with `nx add` (preferred for Material)**

Nx provides `nx add` which runs the schematic so Material's theme files get scaffolded:

```bash
cd frontend
pnpm nx add @angular/material@18.2 --no-interactive
cd ..
```

This installs `@angular/material` + `@angular/cdk` and updates `apps/web-portal/src/styles.scss` with Material theme imports + `apps/web-portal/index.html` body classes. It usually only modifies the FIRST app it finds (web-portal). Apply manually to admin-cms if needed (Step 4).

- [ ] **Step 2: Install Bootstrap grid and other packages**

```bash
cd frontend
pnpm add bootstrap@5.3.3 @ngx-translate/core@16 @ngx-translate/http-loader@16 angular-auth-oidc-client@18
cd ..
```

- [ ] **Step 3: Wire Bootstrap grid into both apps**

Bootstrap is grid + utilities ONLY — DO NOT import the full Bootstrap CSS (would clash with Material). Add to each app's `src/styles.scss` (top of file):

```scss
// Bootstrap 5 — grid + utilities ONLY (not components, per ADR-0003).
@use "bootstrap/scss/functions";
@use "bootstrap/scss/variables";
@use "bootstrap/scss/maps";
@use "bootstrap/scss/mixins";
@use "bootstrap/scss/utilities";
@use "bootstrap/scss/grid";
@use "bootstrap/scss/utilities/api";
```

This pulls only what's needed for layout. CSS bundle delta: ~15 KB compared to full Bootstrap's ~190 KB.

- [ ] **Step 4: Apply Material theme to admin-cms (since `nx add` only touched web-portal)**

Copy the Material theme block from `frontend/apps/web-portal/src/styles.scss` to `frontend/apps/admin-cms/src/styles.scss` (above the Bootstrap block). It looks like:

```scss
@use "@angular/material" as mat;
@include mat.elevation-classes();
@include mat.app-background();

$cce-primary: mat.define-palette(mat.$indigo-palette);
$cce-accent: mat.define-palette(mat.$pink-palette);
$cce-warn: mat.define-palette(mat.$red-palette);
$cce-theme: mat.define-light-theme(
  (
    color: (
      primary: $cce-primary,
      accent: $cce-accent,
      warn: $cce-warn,
    ),
  )
);

@include mat.all-component-themes($cce-theme);
```

Phase 10 replaces the placeholder Material palette with DGA tokens.

- [ ] **Step 5: Build to verify**

```bash
cd frontend
pnpm nx run-many -t build --projects=web-portal,admin-cms --skip-nx-cache 2>&1 | tail -10
cd ..
```

Expected: both build, CSS bundle includes Material + Bootstrap grid utilities.

```bash

git -c commit.gpgsign=false commit -m "feat(phase-09): install Angular Material 18 + Bootstrap 5 grid + ngx-translate + angular-auth-oidc-client"
```

````markdown
# CCE Frontend

Nx 20 monorepo with two Angular 18 apps — `web-portal` (public, port 4200) and `admin-cms` (admin, port 4201).

## Prerequisites

- Node 20.x or 22.x LTS (`nvm use` reads `.nvmrc`)
- pnpm 9.x (via `corepack enable && corepack prepare pnpm@9 --activate`)

## Bootstrap

```bash
cd frontend
pnpm install
```
````

## Common commands

| Command                                                               | What it does                               |
| --------------------------------------------------------------------- | ------------------------------------------ |
| `pnpm nx serve web-portal`                                            | Dev server on http://localhost:4200        |
| `pnpm nx serve admin-cms`                                             | Dev server on http://localhost:4201        |
| `pnpm nx build web-portal`                                            | Production build to `dist/apps/web-portal` |
| `pnpm nx test web-portal --watch=false`                               | Run Jest tests once                        |
| `pnpm nx lint web-portal`                                             | ESLint + Prettier check                    |
| `pnpm nx e2e web-portal-e2e`                                          | Playwright E2E (Phase 14 fills these)      |
| `pnpm nx run-many -t build,lint,test --projects=web-portal,admin-cms` | Full smoke                                 |
| `pnpm nx graph`                                                       | Visualize project dependency graph         |

## Stack

- **Framework:** Angular 18.2 (esbuild bundler, SCSS, no SSR)
- **UI:** Angular Material 18 + Bootstrap 5 grid (utilities only — no Bootstrap components)
- **i18n:** `@ngx-translate/core` + `@ngx-translate/http-loader`
- **Auth:** `angular-auth-oidc-client` (Keycloak code-flow + PKCE)
- **Test:** Jest (unit) + Playwright (E2E with axe-core a11y, Phase 14)
- **Lint:** ESLint with `@angular-eslint/template` a11y rules + Prettier

## What's added in later phases

- Phase 10: Shared libs (`ui-kit`, `i18n`, `auth`, `api-client`, `contracts`)
- Phase 11: web-portal shell (header, locale switcher, router, /health page)
- Phase 12: admin-cms shell + OIDC login flow
- Phase 13: OpenAPI client generation pipeline
- Phase 14: Playwright + axe-core E2E coverage

